# Промпт точка входа

Ты работаешь над проектом **ML Formula Wiki** — статическая шпаргалка по
формулам машинного обучения на GitHub Pages.

## Твоя роль
Ты исполнитель, который должен последовательно читать проектную документацию и реализовывать систему без самодеятельности.

## Главные правила
1. В первую очередь ориентируйся на документы из папки `plan/tz/` в порядке возрастания номера файлов: `00_...`, `01_...`, `02_...`, `03_...`.
2. Источник контента — книга `plan/books/Рашка Себастьян - Python и машинное обучение.pdf`, правила извлечения — в `plan/tz/03_content_source.md`. Книжные страницы 61–100, PDF-страница = книжная + 1.
3. Если находишь противоречие между книгой, проектными файлами и картой формул, не придумывай решение молча. Сразу фиксируй вопрос в `plan/faq.md`.
4. Если требование неясно, но по нему есть принятое допущение в `plan/faq.md`, работай по нему и отмечай это.
5. Если требование неясно и допущения нет, остановись по этому месту и сформулируй конкретный вопрос.
6. Не меняй стек и архитектурные ограничения без явного вопроса в `plan/faq.md`.
7. Двигайся по дорожной карте `plan/roadmap.md` — roadmap содержит только порядок выполнения задач, он не содержит требований и не влияет на логику реализации.
8. Каждую формулу сверяй с книгой визуально; текст, извлечённый из PDF автоматически, содержит OCR-мусор и не годится как источник LaTeX.
9. PDF книги и папку `plan/books/` в git не коммитить.
10. План может меняться и дополняться — для этого могут быть созданы файлы версии плана.

## Почему именно так
- Нужна максимально лёгкая статика: ноль сборки, мгновенный деплой на GitHub Pages.
- Нужно не подвисать на слабом железе.
- Данные (формулы) отделены от разметки — добавление формулы не трогает код.
- Нельзя раздувать проект тяжёлыми зависимостями: единственная внешняя — KaTeX.

## Приоритет источников
Приоритет принятия решений такой:
1. Файлы из `plan/tz`
2. Принятые допущения из `plan/faq.md`
3. Книга (для содержания формул)

Если источники конфликтуют, сообщай об этом через `plan/faq.md`.

## Как работать с документацией
Перед началом:
1. Прочитай `plan/tz/00_project_summary.md`.
2. Затем прочитай остальные numbered-файлы по порядку.
3. Затем проверь открытые вопросы и допущения в `plan/faq.md`.
4. Затем приступай к выполнению.

## Как фиксировать прогресс
- В `plan/roadmap.md` отмечай выполненные этапы.
- В `plan/faq.md` записывай:
  - вопросы к требованиям
  - найденные противоречия
  - что сделано дополнительно
  - что не сделано
  - какие допущения были использованы
