# 01. Tech stack

> Приоритеты те же, что в точке входа: максимально лёгкая статика, ноль сборки,
> мгновенный деплой на GitHub Pages, никакой тяжести ради 20 карточек.
> Менять стек без вопроса в `plan/faq.md` нельзя.

## Выбор: vanilla-статика без сборщика

- **HTML + CSS + vanilla JS** — один `index.html`, один `styles.css`, один
  `app.js`. Никакого Node-тулчейна, npm и CI-сборки: пуш в ветку → сайт обновлён.
- **Данные отдельно от разметки** — все формулы лежат в `data/formulas.js`
  (массив объектов, схема в `02_data_model.md`); `app.js` рендерит карточки из
  данных. Добавить формулу = добавить объект в массив.
- **KaTeX** (CDN, `auto-render` не нужен — рендерим точечно из данных) —
  рендер LaTeX-формул. Выбран вместо MathJax: в разы быстрее, меньше весит,
  не требует конфигурации.
- Поиск — фильтрация массива на клиенте, без библиотек.
- Иконки/шрифты — системные, без внешних зависимостей кроме KaTeX.

## Хостинг и деплой

- **GitHub Pages** из ветки `main` (корень репозитория или папка `/docs`).
- Репозиторий публичный, кастомный домен не требуется.
- PDF книги в репозиторий **не коммитим** (авторские права + 150 МБ) —
  он остаётся только локально в `plan/books/` (папка в `.gitignore`).

## Структура репозитория

```text
/
├── index.html          — страница-шпаргалка
├── styles.css          — стили (mobile-first)
├── app.js              — рендер карточек, поиск, копирование LaTeX
├── data/
│   └── formulas.js     — все формулы (единственный источник данных)
├── plan/               — проектная документация (books/ в .gitignore)
└── README.md
```

## Почему не генератор (11ty/Astro/Jekyll)

- 20 карточек на одной странице не окупают сборочный пайплайн.
- Vanilla-вариант правится и деплоится с любого железа без установки чего-либо.
- Если проект вырастет (сотни формул, отдельные страницы, другие книги) —
  миграция на генератор проста, потому что данные уже отделены от разметки
  (`data/formulas.js` конвертируется в коллекцию контента механически).

## Чего избегаем

- Фреймворков (React/Vue) и сборщиков — ради одной страницы это балласт.
- MathJax — тяжёлый и медленный для статической шпаргалки.
- Встраивания текста формул картинками — только LaTeX (доступность, копирование).

## Риски

- Извлечение формул из PDF ненадёжно (OCR-мусор в русском издании) — формулы
  сверяются с книгой глазами и набираются в LaTeX вручную, не копипастом.
- KaTeX поддерживает не весь LaTeX — при экзотической записи упрощаем нотацию
  и фиксируем это в `plan/faq.md`.
