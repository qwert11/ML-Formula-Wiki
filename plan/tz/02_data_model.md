# 02. Data model — карточка формулы

Единственная сущность проекта — **Formula**. Все формулы лежат массивом в
`data/formulas.js` в порядке следования в книге.

## Схема Formula

```js
{
  id: "sigmoid",              // слаг: латиница, kebab-case, уникален
  block: "B",                 // блок A–D (см. 00_project_summary.md)
  order: 6,                   // сквозной порядковый номер по книге
  page: 74,                   // страница книги (книжная нумерация)
  chapter: "Глава 3",         // глава книги

  latex: "\\phi(z) = \\frac{1}{1 + e^{-z}}",  // формула в LaTeX (KaTeX-совместимо)

  name: "Сигмоида (логистическая функция)",   // краткое название
  discovered: {
    who: "Пьер Франсуа Ферхюльст",            // кто ввёл/открыл
    year: 1838,                               // год (число или строка "1838–1845")
    note: "введена при моделировании роста популяции" // опционально
  },

  params: [                   // описание каждого символа формулы
    { symbol: "z",        desc: "чистый вход: z = wᵀx, взвешенная сумма признаков" },
    { symbol: "\\phi(z)", desc: "выход в диапазоне (0, 1), интерпретируется как вероятность класса" },
    { symbol: "e",        desc: "основание натурального логарифма" }
  ],

  description: "Детальное описание: что делает формула, зачем нужна, " +
               "как используется в машинном обучении, связь с соседними формулами.",

  examples: [                 // 1–3 реальных примера применения
    "Логистическая регрессия в медицине: вероятность заболевания по симптомам пациента.",
    "Скоринг в банках: вероятность дефолта заёмщика.",
    "Функция активации нейрона в ранних нейронных сетях."
  ],

  // Разобранный пример хранится отдельно в data/worked-examples.js под тем же
  // id. Это основная поясняющая часть карточки: реальная задача, цель, связь
  // чисел с обозначениями, исходная формула, полная подстановка, шаги с
  // комментариями, смысл результата и практическое применение.

  // ОПЦИОНАЛЬНО: пошаговый вывод формулы (секция «Как выводится»).
  // Есть только у формул, вывод которых дан в книге или очевиден;
  // если поля нет — секция не рендерится.
  derivation: [
    { latex: "\\operatorname{logit}(p) = w^T x = z",
      comment: "приравниваем логит вероятности к линейной модели" },
    { latex: "\\log\\frac{p}{1-p} = z \\;\\Rightarrow\\; \\frac{p}{1-p} = e^{z}",
      comment: "потенцируем обе части" },
    { latex: "p = \\frac{e^{z}}{1+e^{z}} = \\frac{1}{1+e^{-z}}",
      comment: "выражаем p — получаем сигмоиду" }
  ]
}
```

## Правила заполнения

- `order` — строго по порядку появления в книге; сортировка сайта опирается
  только на это поле.
- `page` — страница первого появления формулы в диапазоне 61–100.
  Соответствие: **PDF-страница = книжная страница + 1**.
- `latex` — набирается вручную и сверяется с книгой; экранирование `\\`
  обязательно (JS-строка). Проверять рендер в KaTeX до коммита.
- `discovered` — по внешним источникам (книга авторство указывает не всегда);
  если авторство спорное или коллективное, писать честно
  («Кортес и Вапник, 1995 — мягкий зазор»), сомнения — в `plan/faq.md`.
- `params` — перечислить **все** символы формулы, включая индексы, если их
  смысл неочевиден. Символы в `symbol` — тоже LaTeX (рендерятся KaTeX).
- `description` — своими словами, 3–8 предложений; не цитировать книгу дословно.
- `examples` — только реальные прикладные сценарии, не «пример: подставим x=2».
- `derivation` — только там, где вывод есть в книге (со сверкой по странице)
  или тривиально восстанавливается (сигмоида из логита). Не выдумывать выводы
  для определений (стандартизация, отношение шансов и т. п.). Каждый шаг —
  одна строка LaTeX + короткий комментарий «что сделали». 3–6 шагов;
  длиннее — значит, это пересказ книги, а не шпаргалка.
