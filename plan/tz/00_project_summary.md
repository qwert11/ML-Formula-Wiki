# 00. Project summary — ML Formula Wiki (шпаргалка по формулам МО)

## Главная идея

Статический сайт-шпаргалка на GitHub Pages с математическими формулами из книги
Себастьяна Рашки «Python и машинное обучение». Формулы извлекаются **по порядку
со страницы 61 по страницу 100** (книжная нумерация) и оформляются как карточки
единого формата. Пользователь открывает сайт, листает/ищет формулу, видит её
запись, смысл, параметры и живые примеры применения.

Ключевая ценность — не «ещё один справочник», а **привязка к конкретной книге**:
формулы идут в том же порядке, что и в тексте, с указанием страницы, поэтому
сайт работает как конспект-компаньон при чтении.

## Формат карточки формулы (обязательные поля)

Каждая формула — одна карточка со строго одинаковым набором полей:

| Поле           | Содержание                                                        |
|----------------|-------------------------------------------------------------------|
| Формула        | Запись в LaTeX (рендер через KaTeX)                               |
| Краткое название | Общепринятое имя (напр. «Сигмоида», «Неоднородность Джини»)     |
| Год и автор    | Кто и когда ввёл/открыл (напр. «К. Шеннон, 1948»)                 |
| Параметры      | Список всех символов формулы с расшифровкой каждого              |
| Что делает     | Детальное описание: зачем формула, как работает, роль в МО       |
| Реальные примеры | 1–3 примера применения на практике (медицина, спам-фильтр и т.д.) |
| Источник       | Страница книги + раздел (напр. «стр. 61, гл. 2»)                 |

## Содержание (карта формул по страницам 61–100)

Диапазон покрывает конец главы 2 и большую часть главы 3. Предварительный
список — в `03_content_source.md`, итоговый уточняется при извлечении.

| Блок | Тема                                   | Страницы книги |
|------|----------------------------------------|----------------|
| A    | Основы обучения: стандартизация, стохастический градиентный спуск | 61–67 |
| B    | Логистическая регрессия: шансы, логит, сигмоида, правдоподобие, функция стоимости, регуляризация L2 | 73–82 |
| C    | Метод опорных векторов: максимальный зазор, мягкий зазор, ядерный трюк, ядро RBF | 84–92 |
| D    | Деревья решений: прирост информации, энтропия, мера Джини, ошибка классификации | 93–100 |

Ожидаемый объём: **~15–20 карточек**.

## Структура сайта

```text
/                — главная: список всех формул (оглавление по блокам A–D)
/#formula-<id>   — якорь на карточку формулы (одностраничник)
```

Одностраничник (SPA без фреймворка): все карточки на одной странице,
сгруппированы по блокам, сверху — оглавление и строка поиска по названию.
Это осознанное упрощение для ~20 карточек; отдельные страницы не нужны.

## Функциональность

- Список формул в порядке следования в книге, группировка по блокам.
- Поиск/фильтр по названию формулы (клиентский, по мере ввода).
- Рендер формул KaTeX (быстрее MathJax, работает статически).
- Копирование LaTeX-записи формулы в буфер по кнопке.
- Адаптивная вёрстка: читабельно на телефоне (шпаргалка = мобильный сценарий).
- Тёмная/светлая тема — опционально (см. `plan/faq.md`).

## Что НЕ делаем (границы)

- Никакого backend, БД, авторизации — только статика на GitHub Pages.
- Не пересказываем книгу: только формулы и их объяснение, без длинных глав.
- Не выкладываем текст книги и её сканы — только формулы (математические факты)
  с собственными описаниями и ссылкой на источник.
- Не берём формулы вне диапазона стр. 61–100 (даже если они «напрашиваются»,
  напр. SSE со стр. 55) — фиксируем такие случаи в `plan/faq.md`.
- Код из книги (Python-реализации) не включаем в карточки на первом этапе.

## Definition of Done

1. Все формулы со стр. 61–100 извлечены, каждая карточка заполнена по всем
   7 полям, ни одно поле не пустое.
2. Формулы корректно рендерятся KaTeX (сверены с книгой визуально).
3. Порядок карточек совпадает с порядком в книге, у каждой указана страница.
4. Поиск фильтрует карточки по названию.
5. Сайт открывается с GitHub Pages по публичному URL и работает на мобильном.
6. README: что это, источник, как добавить формулу, как задеплоить.
